部署Qexo

Hugo 为纯静态博客，不同于 WordPress、Typecho，并无管理后台。一般都是通过本地推送到 github 私有仓库，然后再通过 Action 推送到 GitPages，但是有的时候并不方便，比如：无法使用手机编辑博客，在公司电脑上写博客不方便等

为了能够通过网页编辑 Hugo 博客，可以通过 Qexo 编辑文章内容，并自动推送到 Github 私有仓库，然后再通过 Action 推送到 GitPages，实现博客的网页端编辑

因为 Qexo 创建 Hugo 文章的时候，默认是在 content/posts 里面进行创建，但由于这里使用的主题文章路径为 content/post，所以我们需要对 Qexo 的源码进行修改，直接在 github 克隆一份 Qexo 源码，修改内容如下：

Docker 本地部署

参考：https://haohanxinghe.com/posts/21.html

Qexo 推荐的本地部署方式是直接采用 Python 环境部署，这种方式实际并不方便，所以采用 Docker 部署

首先，新建 Qexo 文件夹，用于存放 Qexo 应用程序和构建镜像所需要的文件

mkdir /www/qexo -p
cd /www/qexo
mkdir build
cd build

进入 buid 文件夹，新增三个文件：Dockerfile，configs.py，run.sh

Dockerfile 文件内容如下（注意下面拉取的是我自己克隆的 Qexo 仓库）：

FROM python:3.11.3-alpine
# 维护者信息
LABEL maintainer="codepzj"
# 设置生产模式环境变量
# ENV APP_ENV production
# pipy源设置
RUN pip config set global.index-url https://mirrors.cloud.tencent.com/pypi/simple/
# 设置时区
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories && \
    apk --no-cache add tzdata && \
    cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime && \
    apk --no-cache del tzdata && \
    apk --no-cache add git && \
    mkdir /app
# 设置工作目录
WORKDIR /app
# 拷贝数据
RUN git clone -b master https://github.com/0xo7/Qexo.git .
# 安装依赖
RUN pip install --upgrade pip
RUN pip install -r requirements.txt
# 拷贝启动脚本
COPY . /app/
RUN chmod +x /app/run.sh
# 暴露端口
EXPOSE 8000
# 挂载目录
VOLUME ["/app/db", "/app/data"]
# 启动django
ENTRYPOINT ["/bin/sh", "/app/run.sh"]

configs.py 内容如下：

import pymysql
import os

DOMAINS = "*"
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join('/app/db', 'db.sqlite3'),
    }
}

run.sh 内容如下：

#!/bin/bash

python3 manage.py makemigrations
python3 manage.py migrate
python3 manage.py runserver 0.0.0.0:8000 --noreload

使用如下命令打包镜像

docker build -t qexo .

然后使用如下命令即可运行 docker 管理后台：

# 创建需要挂载的目录
mkdir -p ./data/blog ./data/qexo/db

# 运行容器
docker run -d \
  --name qexo \
  -p 8000:8000 \
  -v "$(pwd)/data/blog:/app/data" \
  -v "$(pwd)/data/qexo/db:/app/db" \
  -v "/etc/localtime:/etc/localtime:ro" \
  -v "/etc/timezone:/etc/timezone:ro" \
  qexo

为了更加方便的使用 docker 镜像，我们可以把 docker 镜像打包上传到 DockerHub，具体操作如下所示：

docker login    # 输入账号密码进行登录
docker tag 43d03519e1e0 ayuxy/qexo   # 给对应的 docker 镜像打标签
docker push ayuxy/qexo     # 推送到 dockerhub 上面
